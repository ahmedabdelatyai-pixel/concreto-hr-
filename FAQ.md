## 🔐 أسئلة الأمان

### س: هل كلمات المرور محفوظة بأمان؟
**ج:** نعم! نستخدم:
- Bcrypt للتشفير (10 rounds)
- Never stored بكلمة مرور نصية
- توليد salt فريد لكل كلمة مرور

---

### س: هل البيانات معزولة بين العملاء؟
**ج:** نعم، معزولة تماماً:
- كل شركة لها معرف فريد
- جميع الـ queries تفلتر حسب company ID
- لا يمكن لعميل رؤية بيانات عميل آخر

---

### س: هل الـ API محمي من الهجمات؟
**ج:** نعم:
- ✅ Rate Limiting (100 طلب/15 دقيقة)
- ✅ Input Validation
- ✅ JWT Tokens
- ✅ Helmet Middleware
- ✅ CORS محدود

---

### س: ماذا لو تم اختراق API Key؟
**ج:** لا تقلق:
- استخدم `.env.local` و `.env` (في .gitignore)
- استخدم متغيرات البيئة في Vercel
- غيّر المفاتيح فوراً في حالة الاشتباه

## 💾 أسئلة البيانات

### س: كيف أتصل بـ MongoDB?
**ج:** ستحتاج إلى:
1. حساب MongoDB Atlas (مجاني)
2. أنشئ Cluster
3. انسخ Connection String
4. ضعها في MONGODB_URI في `.env`

[تعليقات تفصيلية في LAUNCH_GUIDE.md](./LAUNCH_GUIDE.md)

---

### س: هل يدعم قاعدة بيانات محلية؟
**ج:** نعم يدعم:
- MongoDB Atlas (الموصى به)
- mongoDB محلي (للتطوير)
- حتى database.json (fallback)

---

### س: كيف أعمل backup للبيانات?
**ج:** MongoDB Atlas يوفر:
- automatic backups (يومي)
- point-in-time restore
- snapshots يدوية

## 🐛 أسئلة الأخطاء

### س: الخادم لا يبدأ!
**ج:** تحقق:
1. PORT لم يكن مستخدم: `lsof -i :5000`
2. Node version: `node --version` (يجب v18+)
3. المكتبات: `npm install`

---

### س: خطأ "CORS error"!
**ج:** تحقق:
```env
# في server/.env
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
```

---

### س: "MongoDB connection failed"!
**ج:** تحقق:
1. MONGODB_URI صحيح
2. MongoDB يعمل
3. IP في allowlist (MongoDB Atlas)

---

### س: "Invalid token"!
**ج:** السبب:
1. انتهت صلاحية الـ token
2. JWT_SECRET تغير
3. Token مشفر خاطئ

الحل: امسح localStorage وأعد تسجيل الدخول
